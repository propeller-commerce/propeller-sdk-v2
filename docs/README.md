# propeller-sdk-v2 documentation site

This directory is a self-contained [Docusaurus 3](https://docusaurus.io) app.
It is **not** published to npm and is **not** part of the SDK build — it has
its own `package.json` / lockfile and `node_modules`.

Published at <https://propeller-commerce.github.io/propeller-sdk-v2/> by
`.github/workflows/docs.yml` (GitHub Pages).

## Layout

```
docs/
  docusaurus.config.ts   site config (navbar, footer, branding, search)
  sidebars.ts            manual guide tree + autogenerated API Reference
  content/               documentation pages (Docusaurus docs root)
    **/*.md               hand-authored guides (sourced from the repo README)
    api/                  GENERATED API reference (gitignored)
    changelog.md          GENERATED copy of root CHANGELOG.md (gitignored)
  src/                   landing page + theme overrides (custom.css)
  static/img/            branding assets (PLACEHOLDERS — see static/img/README.md)
  scripts/
    gen-api.js            standalone TypeDoc -> content/api (+ typedoc-sidebar.cjs)
    sanitize-api-mdx.js   makes generated markdown MDX-safe (run after gen-api)
    copy-changelog.js     copies root CHANGELOG.md -> content/changelog.md
    history-codemods/     provenance: spent one-shot codemods (not run)
```

## Local development

From the repo root, first build the SDK's generated GraphQL barrel
(`src/index.ts` re-exports it and TypeDoc needs it):

```bash
npm run build:graphql
```

Then, in this directory:

```bash
npm install      # first time
npm start        # dev server at http://localhost:3000/propeller-sdk-v2/
npm run build    # production build into docs/build
npm run serve    # serve the production build
npm run typecheck
```

`prestart` / `prebuild` automatically run `gen:changelog` and `gen:api`
(TypeDoc + sanitize) so the API reference and changelog exist before the site
builds.

## Why TypeDoc is decoupled from the build

The SDK API reference is generated by **standalone TypeDoc**
(`scripts/gen-api.js`) and made MDX-safe by `scripts/sanitize-api-mdx.js`
*before* `docusaurus build` runs. We deliberately do **not** use the
`docusaurus-plugin-typedoc` build plugin:

- The plugin regenerates `content/api/**` on every `docusaurus build`,
  clobbering the on-disk sanitize pass.
- MDX v3 fails to parse raw TypeDoc output (bare `Promise<Foo>` / `{x}` in
  JSDoc prose) *before* any remark plugin can transform it, so an in-build
  remark fix is impossible.

Decoupling lets the pipeline be: `build:graphql → gen-api → sanitize →
docusaurus build`, with the API treated as ordinary pre-sanitized static
markdown.

## Search

Local offline search via `@easyops-cn/docusaurus-search-local`. To switch to
Algolia DocSearch once credentials exist:

1. Remove the `@easyops-cn/docusaurus-search-local` plugin entry from
   `docusaurus.config.ts`.
2. Add `algolia: { appId, apiKey, indexName, contextualSearch: true }` to
   `themeConfig` (the classic preset already ships the DocSearch component).

## Branding

`static/img/*` are placeholders. See `static/img/README.md` for the required
assets and dimensions, and `src/css/custom.css` for the placeholder brand
color (`--ifm-color-primary`).
